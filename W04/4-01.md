# Предпоставки / Сценарий

Двама служители се нуждаят от достъп до услуги, предоставяни от сървъра. **PC1** се нуждае единствено от FTP достъп, а **PC2** — единствено от уеб достъп. И двата компютъра трябва да могат да изпращат ping до сървъра, но не и един до друг.

# Инструкции

## Част I: Конфигуриране, прилагане и проверка на разширен номериран ACL

### Стъпка 1: Конфигурирайте ACL за разрешаване на FTP и ICMP трафик от LAN мрежата на PC1.

а. От режима на глобална конфигурация на **R1** въведете следната команда, за да определите първия валиден номер за разширен списък за достъп.

*Отворете прозореца за конфигурация.*

```
R1(config)# access-list ?
  <1-99>    IP стандартен списък за достъп
  <100-199> IP разширен списък за достъп
```

б. Добавете **100** към командата, последвано от въпросителен знак.

```
R1(config)# access-list 100 ?
  deny   Задайте пакети за отхвърляне
  permit Задайте пакети за препращане
  remark Коментар към запис в списъка за достъп
```

в. За да разрешите FTP трафик, въведете **permit**, последвано от въпросителен знак.

```
R1(config)# access-list 100 permit ?
  ahp   Протокол за удостоверяване на заглавия (AHP)
  eigrp Протокол за маршрутизация EIGRP на Cisco
  esp   Капсулиране на полезния товар за защита (ESP)
  gre   Тунелиране GRE на Cisco
  icmp  Интернет протокол за управляващи съобщения (ICMP)
  ip    Произволен интернет протокол
  ospf  Протокол за маршрутизация OSPF
  tcp   Протокол за управление на предаването (TCP)
  udp   Протокол за потребителски дейтаграми (UDP)
```

г. При конфигуриране и прилагане, този ACL трябва да разреши FTP и ICMP трафик. ICMP е изброен по-горе, но FTP — не. Това е така, защото FTP е протокол на приложния слой, който използва TCP на транспортния слой. Въведете **tcp**, за да прецизирате допълнително помощната информация на ACL.

```
R1(config)# access-list 100 permit tcp ?
  A.B.C.D Адрес на източника
  any     Произволен хост източник
  host    Единичен хост източник
```

д. Адресът на източника може да представлява едно устройство, например PC1, чрез ключовата дума **host** и след нея IP адреса на PC1. Ключовата дума **any** разрешава произволен хост от произволна мрежа. Филтрирането може да се извърши и по мрежов адрес. В случая — всеки хост с адрес, принадлежащ към мрежа 172.22.34.64/27. Въведете този мрежов адрес, последван от въпросителен знак.

```
R1(config)# access-list 100 permit tcp 172.22.34.64 ?
  A.B.C.D Инверсни битове на маската на източника
```

е. Изчислете инверсната маска, като определите двоичното допълнение на подмрежовата маска /27.

```
11111111.11111111.11111111.11100000 = 255.255.255.224
00000000.00000000.00000000.00011111 = 0.0.0.31
```

ж. Въведете инверсната маска, последвана от въпросителен знак.

```
R1(config)# access-list 100 permit tcp 172.22.34.64 0.0.0.31 ?
  A.B.C.D Адрес на получателя
  any     Произволен хост получател
  eq      Съвпадение само на пакети на даден номер на порт
  gt      Съвпадение само на пакети с по-висок номер на порт
  host    Единичен хост получател
  lt      Съвпадение само на пакети с по-нисък номер на порт
  neq     Съвпадение само на пакети, които не са на даден номер на порт
  range   Съвпадение само на пакети в диапазон от номера на портове
```

з. Конфигурирайте адреса на получателя. В този сценарий филтрираме трафика към единичен получател — сървъра. Въведете ключовата дума **host**, последвана от IP адреса на сървъра.

```
R1(config)# access-list 100 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 ?
  dscp        Съвпадение на пакети с даден DSCP код
  eq          Съвпадение само на пакети на даден номер на порт
  established Установена TCP сесия
  gt          Съвпадение само на пакети с по-висок номер на порт
  lt          Съвпадение само на пакети с по-нисък номер на порт
  neq         Съвпадение само на пакети, които не са на даден номер на порт
  precedence  Съвпадение на пакети с даден приоритет
  range       Съвпадение само на пакети в диапазон от номера на портове
  <cr>
```

и. Забележете, че една от опциите е **\<cr\>** (връщане на каретката). С други думи, можете да натиснете **Enter** и записът ще разреши целия TCP трафик. Тъй като разрешаваме само FTP трафик, въведете ключовата дума **eq**, последвана от въпросителен знак, за да видите наличните опции, след което въведете **ftp** и натиснете **Enter**.

```
R1(config)# access-list 100 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 eq ?
  <0-65535> Номер на порт
  ftp       Протокол за прехвърляне на файлове (21)
  pop3      Протокол на пощенската кутия v3 (110)
  smtp      Прост протокол за пренос на поща (25)
  telnet    Telnet (23)
  www       Световна мрежа (HTTP, 80)

R1(config)# access-list 100 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 eq ftp
```

й. Създайте втори запис в списъка за достъп, за да разрешите ICMP трафик (ping и др.) от PC1 до сървъра. Забележете, че номерът на списъка за достъп остава същият и не е необходимо да се указва конкретен вид ICMP трафик.

```
R1(config)# access-list 100 permit icmp 172.22.34.64 0.0.0.31 host 172.22.34.62
```

к. Целият останал трафик се блокира по подразбиране.

л. Изпълнете командата **show access-list** и проверете дали списък за достъп 100 съдържа правилните записи. Забележете, че записът **deny any any** не се появява в края на списъка. Поведението по подразбиране е такова, че ако пакетът не съответства на нито един запис в списъка, той не се пропуска през интерфейса.

```
R1# show access-lists
Extended IP access list 100
    10 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 eq ftp
    20 permit icmp 172.22.34.64 0.0.0.31 host 172.22.34.62
```

### Стъпка 2: Приложете ACL към правилния интерфейс за филтриране на трафика.

От гледна точка на **R1** трафикът, към който се прилага ACL 100, е входящ от мрежата, свързана към интерфейс Gigabit Ethernet 0/0. Влезте в режима на конфигурация на интерфейса и приложете ACL.

> **Забележка:** В реална оперативна мрежа не се препоръчва прилагането на нетестван списък за достъп към активен интерфейс.

```
R1(config)# interface gigabitEthernet 0/0
R1(config-if)# ip access-group 100 in
```

### Стъпка 3: Проверете реализацията на ACL.

а. Изпратете ping от **PC1** до **Server**. Ако ping заявките не са успешни, проверете IP адресите, преди да продължите.

б. Осъществете FTP връзка от **PC1** до **Server**. Потребителското име и паролата са **cisco**.

```
PC> ftp 172.22.34.62
```

в. Излезте от FTP услугата.

```
ftp> quit
```

*Затворете прозореца за конфигурация.*

г. Изпратете ping от **PC1** до **PC2**. Хостът получател трябва да е недостъпен, тъй като ACL не е разрешил изрично този трафик.

## Част II: Конфигуриране, прилагане и проверка на разширен именуван ACL

### Стъпка 1: Конфигурирайте ACL за разрешаване на HTTP достъп и ICMP трафик от LAN мрежата на PC2.

а. Именуваните ACL започват с ключовата дума **ip**. От режима на глобална конфигурация на **R1** въведете следната команда, последвана от въпросителен знак.

*Отворете прозореца за конфигурация.*

```
R1(config)# ip access-list ?
  extended Разширен списък за достъп
  standard Стандартен списък за достъп
```

б. Можете да конфигурирате стандартни и разширени именувани ACL списъци. Тъй като този списък за достъп филтрира едновременно IP адресите на източника и получателя, той трябва да бъде разширен. Въведете **HTTP_ONLY** като наименование. (За оценяване в Packet Tracer наименованието е чувствително към регистъра на буквите и записите трябва да бъдат в правилен ред.)

```
R1(config)# ip access-list extended HTTP_ONLY
```

в. Подканващият знак се променя — вече сте в режим на конфигурация на разширен именуван ACL. Всички устройства в LAN мрежата на **PC2** се нуждаят от TCP достъп. Въведете мрежовия адрес, последван от въпросителен знак.

```
R1(config-ext-nacl)# permit tcp 172.22.34.96 ?
  A.B.C.D Инверсни битове на маската на източника
```

г. Алтернативен начин за изчисляване на инверсната маска е да се извади подмрежовата маска от 255.255.255.255.

```
  255.255.255.255
- 255.255.255.240
─────────────────
=   0.  0.  0. 15

R1(config-ext-nacl)# permit tcp 172.22.34.96 0.0.0.15
```

д. Завършете записа, като зададете адреса на сървъра по същия начин като в Част I, и филтрирайте **www** трафика.

```
R1(config-ext-nacl)# permit tcp 172.22.34.96 0.0.0.15 host 172.22.34.62 eq www
```

е. Създайте втори запис в списъка за достъп, за да разрешите ICMP трафик (ping и др.) от **PC2** до **Server**. Забележете: подканващият знак остава същият и не е необходимо да се указва конкретен вид ICMP трафик.

```
R1(config-ext-nacl)# permit icmp 172.22.34.96 0.0.0.15 host 172.22.34.62
```

ж. Целият останал трафик се блокира по подразбиране. Излезте от режима на конфигурация на разширен именуван ACL.

з. Изпълнете командата **show access-list** и проверете дали списък за достъп **HTTP_ONLY** съдържа правилните записи.

```
R1# show access-lists
Extended IP access list 100
    10 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 eq ftp
    20 permit icmp 172.22.34.64 0.0.0.31 host 172.22.34.62
Extended IP access list HTTP_ONLY
    10 permit tcp 172.22.34.96 0.0.0.15 host 172.22.34.62 eq www
    20 permit icmp 172.22.34.96 0.0.0.15 host 172.22.34.62
```

### Стъпка 2: Приложете ACL към правилния интерфейс за филтриране на трафика.

От гледна точка на **R1** трафикът, към който се прилага списък за достъп **HTTP_ONLY**, е входящ от мрежата, свързана към интерфейс Gigabit Ethernet 0/1. Влезте в режима на конфигурация на интерфейса и приложете ACL.

> **Забележка:** В реална оперативна мрежа не се препоръчва прилагането на нетестван списък за достъп към активен интерфейс. Това трябва да се избягва, когато е възможно.

```
R1(config)# interface gigabitEthernet 0/1
R1(config-if)# ip access-group HTTP_ONLY in
```

### Стъпка 3: Проверете реализацията на ACL.

а. Изпратете ping от **PC2** до **Server**. Ако ping заявката не е успешна, проверете IP адресите, преди да продължите.

б. От **PC2** отворете уеб браузър и въведете IP адреса на сървъра. Уеб страницата на сървъра трябва да се зареди.

в. Осъществете FTP връзка от **PC2** до **Server**. Връзката трябва да бъде неуспешна. Ако не е, диагностицирайте записите в списъка за достъп и конфигурациите на **access-group** на интерфейсите.

*Затворете прозореца за конфигурация.*

## Речник на термините

| Термин (английски) | Термин (български) | Пояснение |
|---|---|---|
| Extended ACL | Разширен ACL | Списък за достъп, филтриращ трафика по IP адрес на източника, IP адрес на получателя, протокол и номер на порт |
| Standard ACL | Стандартен ACL | Списък за достъп, филтриращ трафика само по IP адрес на източника |
| Numbered ACL | Номериран ACL | ACL, идентифициран с число (1–99 за стандартен, 100–199 за разширен) |
| Named ACL | Именуван ACL | ACL, идентифициран с описателно наименование вместо с число |
| Wildcard mask | Инверсна маска | Маска, обратна на подмрежовата маска; използва се в ACL за определяне на обхвата на адресите |
| permit | разреши | Правило в ACL, което пропуска съответния трафик |
| deny | блокирай / забрани | Правило в ACL, което отхвърля съответния трафик |
| FTP (File Transfer Protocol) | Протокол за прехвърляне на файлове | Протокол от приложния слой за трансфер на файлове; използва TCP порт 21 |
| HTTP (HyperText Transfer Protocol) | Протокол за пренос на хипертекст | Протокол за уеб комуникация; използва TCP порт 80 |
| ICMP (Internet Control Message Protocol) | Интернет протокол за управляващи съобщения | Протокол, използван за диагностика (ping) и управление на мрежи |
| TCP (Transmission Control Protocol) | Протокол за управление на предаването | Протокол от транспортния слой с гарантирана доставка |
| inbound / incoming | входящ | Трафик, постъпващ в интерфейса на маршрутизатора от мрежата |
| outbound / outgoing | изходящ | Трафик, напускащ интерфейса на маршрутизатора към мрежата |
| access-group | група за достъп | Команда за свързване на ACL с конкретен интерфейс и посока |
| LAN (Local Area Network) | Локална мрежа | Мрежа, обхващаща ограничена географска област |
| NIC (Network Interface Card) | Мрежова карта | Хардуерен компонент, осигуряващ мрежова свързаност на устройството |
| Default gateway | Шлюз по подразбиране | Маршрутизаторът, към който се изпраща трафикът, предназначен за отдалечени мрежи |
| DSCP (Differentiated Services Code Point) | Код за диференцирани услуги | Поле в IP заглавието за маркиране на приоритета на трафика |
| eq (equal) | равно на | Оператор в ACL за филтриране по конкретен номер на порт |
| show run | show run | Команда за показване на текущата работна конфигурация на устройството |
| show access-lists | show access-lists | Команда за показване на всички конфигурирани ACL списъци |
