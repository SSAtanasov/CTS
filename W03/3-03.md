# Предпоставки

В тази дейност ще наблюдавате как един списък за управление на достъпа (ACL) може да се използва за блокиране на ICMP ехо заявки (ping) до възли в отдалечени мрежи. След като ACL бъде премахнат от конфигурацията, ping заявките ще бъдат успешни.

## Част II: Премахване на ACL и повторно тестване

### Стъпка 1: Използвайте show команди, за да проучите конфигурацията на ACL.

а. Използвайте командите **show run** и **show access-lists**, за да прегледате текущо конфигурираните ACL списъци. За бърз преглед на ACL използвайте **show access-lists**. Въведете командата **show access-lists**, последвана от интервал и въпросителен знак (?), за да видите наличните опции:

```
R1# show access-lists ?
  <1-199>  Номер на ACL
  WORD     Наименование на ACL
  <cr>
```

Ако знаете номера или наименованието на ACL, можете да филтрирате изхода на командата **show** допълнително. Тъй като **R1** има само един ACL, командата **show access-lists** е достатъчна.

```
R1# show access-lists
Standard IP access list 11
    10 deny   192.168.10.0 0.0.0.255
    20 permit any
```

Първият ред на ACL блокира всички пакети, произхождащи от мрежа **192.168.10.0/24**, включително ICMP ехо заявки (ping). Вторият ред разрешава целия останал IP трафик от произволен източник да преминава през маршрутизатора.

б. За да влияе ACL върху работата на маршрутизатора, той трябва да бъде приложен към даден интерфейс в определена посока. В този сценарий ACL се използва за филтриране на трафика, напускащ интерфейса. Следователно целият трафик, излизащ от посочения интерфейс на R1, ще бъде проверяван спрямо ACL 11.

Въпреки че можете да видите IP информация чрез командата **show ip interface**, в някои случаи е по-удобно да използвате директно командата **show run**.

### Стъпка 2: Премахнете списък за достъп 11 от конфигурацията.

ACL списъците могат да бъдат премахнати от конфигурацията чрез командата **no access-list** [*номер на ACL*]. Командата **no access-list** без аргумент изтрива всички ACL списъци, конфигурирани на маршрутизатора. Командата **no access-list** [*номер на ACL*] премахва само конкретния ACL.

а. В конфигурацията на интерфейс Serial0/0/0 премахнете access-list 11, приложен към интерфейса като **изходящ** филтър:

```
R1(config)# int se0/0/0
R1(config-if)# no ip access-group 11 out
```

б. В режим на глобална конфигурация премахнете ACL, като въведете следната команда:

```
R1(config)# no access-list 11
```

в. Проверете дали **PC1** вече може да изпрати ping до **DNS сървъра** и до **PC4**.

## Речник на термините

| Термин (английски) | Термин (български) | Пояснение |
|---|---|---|
| Access Control List (ACL) | Списък за управление на достъпа | Набор от правила, дефиниращи кой трафик се разрешава или блокира на даден интерфейс |
| ping | ping / ICMP ехо заявка | Команда за проверка на мрежова свързаност чрез изпращане на ICMP ехо пакети |
| ICMP (Internet Control Message Protocol) | Интернет протокол за управляващи съобщения | Протокол, използван за диагностика и управление на мрежи |
| deny | забрани / блокирай | Правило в ACL, което отхвърля съответния трафик |
| permit | разреши | Правило в ACL, което пропуска съответния трафик |
| wildcard mask | инверсна маска | Маска, обратна на подмрежовата маска, използвана в ACL за определяне на обхвата на адресите |
| outbound / outgoing filter | изходящ филтър | ACL, приложен в посока от маршрутизатора към мрежата |
| global configuration mode | режим на глобална конфигурация | Режим на Cisco IOS, в който се прилагат настройки за целия маршрутизатор |
| interface configuration mode | режим на конфигурация на интерфейс | Режим на Cisco IOS за настройка на конкретен мрежови интерфейс |
| show run | show run | Команда за показване на текущата работна конфигурация на устройството |
| DNS Server | DNS сървър | Сървър за система за имена на домейни, превеждащ имена в IP адреси |
| remote network | отдалечена мрежа | Мрежа, намираща се извън локалния мрежови сегмент |
| simulation mode | режим на симулация | Режим в Cisco Packet Tracer за стъпково проследяване на движението на пакети |
